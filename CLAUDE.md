## Project

**Lexio** is a desktop English learning app. Students search for English words and receive meaning, phonetics, examples and synonyms generated by the GROQ API. Words are cached locally in SQLite.

## Security

- **API key** — The Groq API key is user-managed (stored in SQLite via the settings page). It must NEVER be hardcoded, logged, or exposed in renderer console output.
- **API call location** — `ai.ts` in the renderer is the only place where external API calls are made. The key is retrieved from SQLite via `window.lexio.*` IPC and passed to `ai.ts` — it must not be stored in React state or logged.
- **No secrets in code** — Never commit `.env` files or any file containing a real API key. Treat any key that appears in a diff as compromised.
- **contextBridge** — Only expose the minimum surface needed. Do not add `require`/`shell`/`fs` to the preload bridge.

## Word Search Flow

```
[renderer] hook useSearch
  → window.lexio.getWord("churn")   → Main → SQLite
    If found in DB: returns immediately (no API cost)
    If not found:
      → fetchWordFromGroq("churn") → Groq API
      → window.lexio.saveWord(result) → Main → SQLite upsert
      → returns complete Word
```

## Architecture — 3 Processes

```
MAIN  (Node.js) — src/main/
  ├── index.ts     app lifecycle, BrowserWindow
  ├── db.ts        ALL SQLite logic (better-sqlite3, synchronous)
  ├── ipc.ts       registers handlers → delegates to db.ts
  ├── shortcut.ts  Ctrl+Shift+E — toggle visibility
  └── tray.ts      system tray icon

PRELOAD — src/preload/index.ts
  └── contextBridge.exposeInMainWorld('lexio', api: LexioAPI)

RENDERER (React/Vite) — src/renderer/
  ├── hooks/       state logic, calls to window.lexio.*
  ├── components/  RENDERING ONLY, zero business logic
  └── lib/
      └── ai.ts   ONLY external fetch (Groq API) — no other file may call fetch()
```

**Golden rule:** The renderer NEVER accesses Node.js, filesystem, or SQLite directly. Everything via `window.lexio.*`.

---

## Code Rules

- **TypeScript strict** — zero `any`. Explicit interfaces for everything.
- **Files .ts/.tsx** — max 150 lines. If exceeded, split into modules.
- **Hooks** — all calls to `window.lexio.*` go in `src/renderer/hooks/`
- **DB** — all SQL queries go in `src/main/db.ts`. Never in `ipc.ts`.
- **Serialization** — `better-sqlite3` is synchronous. Arrays (`examples`, `synonyms`, `contexts`) are stored as JSON strings and require `serialize()`/`deserialize()`.
